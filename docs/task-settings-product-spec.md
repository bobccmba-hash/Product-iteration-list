# 配置中心 / 任务设定 — 产品文档

> 版本：1.9.0 | 路由：`/admin/config/tasks`

---

## 一、功能概述

任务设定用于配置下发给学生的短期挑战任务。任务由后台管理员创建后，通过分发策略自动推送给符合条件的学生终端，终端本地按规则计算进度，达成后发放奖励。

**核心逻辑**：任务 = 周期 × 统计指标 × 目标数值。管理员选择统计哪个指标、在什么周期内达到多少数值，即构成一条完整的任务规则。

---

## 二、任务列表页

**路由**：`/admin/config/tasks`

### 2.1 筛选条件

| 字段 | 类型 | 说明 |
|------|------|------|
| 任务名称/编码 | 文字搜索 | 模糊匹配任务名称或编码 |
| 状态 | 下拉 | 全部 / 草稿 / 已发布 / 已停用 |
| 周期 | 下拉 | 全部 / 日 / 周 / 一次性 |

### 2.2 列表字段

| 字段 | 说明 |
|------|------|
| 任务名称 | 任务名称 + 下方显示任务编码（小字 mono 字体） |
| 周期 | 日 / 周 / 一次性，Tag 样式展示 |
| 完成条件 | 统计指标 + 目标数值的摘要描述 |
| 奖励 | 奖励类型 + 数量的摘要描述 |
| 分发策略 | 所有学生 / 新学生默认分配 / 老学生默认分配 |
| 状态 | 草稿（黄）/ 已发布（绿）/ 已停用（红）|
| 更新时间 | 最近一次修改时间 |
| 操作 | 编辑 / 复制 / 推送终端 / 停用·启用 / 删除 |

### 2.3 操作说明

| 操作 | 说明 |
|------|------|
| 编辑 | 进入任务编辑页 |
| 复制 | 复制当前任务为新草稿，名称自动加"_副本" |
| 推送终端 | 跳转至推送终端页，配置下发范围 |
| 停用 | 停用后该任务不再参与新终端分发；确认弹窗提示 |
| 启用 | 将已停用任务重新设为已发布 |
| 删除 | 仅草稿任务可删除；非草稿任务点击删除会提示错误 |

## 三、新建 / 编辑任务页

**路由**：
- 新建：`/admin/config/tasks/new`
- 编辑：`/admin/config/tasks/[taskId]/edit`

两个页面结构完全一致，单页完成所有配置，**不分步骤**。页面从上到下分为五个区块：

---

### 3.1 基本信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 任务名称 | 是 | 展示给管理员的任务标识，建议描述任务目标，如「今天完成 3 次互动」 |
| 任务说明 | 否 | 补充说明任务背景或提示，多行文本 |

---

### 3.2 任务周期

决定统计窗口的重置频率，三选一（卡片单选）：

| 选项 | 值 | 说明 |
|------|-----|------|
| 日 | `daily` | 统计窗口为自然日，每天重置，可反复完成 |
| 周 | `weekly` | 统计窗口为自然周，每周重置，可反复完成 |
| 一次性 | `one_time` | 统计窗口不重置，累计达标后终身完成 |

**示例**：
- 日 × 互动次数 × 3 = 今天互动 3 次即完成，明天重置
- 周 × 不同卡牌数 × 5 = 本周收集 5 种不同卡牌即完成，下周重置
- 一次性 × 连续互动天数 × 7 = 累计连续互动 7 天，终身完成一次

---

### 3.3 完成条件

当统计值达到目标数值时，任务判定为完成。达成公式：`统计指标 >= 目标数值`（在对应周期窗口内）。

**统计指标**（下拉选择）：

| 指标 | 计算口径 |
|------|----------|
| 互动次数 | 选定周期内的互动结束事件数量 |
| 卡牌总数 | 选定周期内获得的卡牌张数（含重复） |
| 不同卡牌数 | 选定周期内去重后的卡牌种类数量 |
| 连续互动天数 | 从最近一次起，连续有互动记录的自然日天数 |
| 连续得卡天数 | 从最近一次起，连续有获得卡牌事件的自然日天数 |
| 任务完成数 | 成长记录中的任务完成事件次数 |

**目标数值**：数字输入，最小值为 1。页面实时显示达成公式，如：`互动次数 >= 3（日）`。

---

### 3.4 奖励

任务完成后发放的奖励。

| 字段 | 类型 | 说明 |
|------|------|------|
| 奖励类型 | 单选（卡片） | 卡牌 / 盲盒 |
| 奖励数量 | 数字输入 | 最小值为 1 |

---

### 3.5 分发策略

配置学生什么时候拿到这个任务，三选一（卡片单选）：

| 选项 | 值 | 说明 |
|------|-----|------|
| 所有学生 | `all` | 对所有学生默认分配，包括新学生和老学生 |
| 新学生默认分配 | `new_default` | 首次建立档案后默认拿到该任务 |
| 老学生默认分配 | `existing_default` | 已有档案但无任务时默认分配 |

---

### 3.6 底部操作

| 按钮 | 说明 |
|------|------|
| 保存配置 | 校验必填项（任务名称、目标数值 >= 1），通过后保存并返回列表 |
| 返回 | 不保存，直接返回任务列表 |

**校验规则**：
- 任务名称为空 → 提示「请填写任务名称」
- 目标数值 < 1 → 提示「目标数值需大于 0」

---

## 四、字段默认值

新建任务时各字段默认值如下：

| 字段 | 默认值 |
|------|--------|
| 任务名称 | 空 |
| 任务说明 | 空 |
| 任务周期 | 日（daily） |
| 统计指标 | 互动次数 |
| 目标数值 | 3 |
| 奖励类型 | 卡牌 |
| 奖励数量 | 1 |
| 分发策略 | 所有学生 |

---

## 五、任务状态流转

```
新建 → 草稿（draft）
草稿 → 已发布（published）  [发布操作]
已发布 → 已停用（disabled）  [停用操作]
已停用 → 已发布（published）  [启用操作]
草稿 → 删除               [仅草稿可删除]
```

## 六、典型配置示例

### 示例 1：每日打卡任务

```
任务名称：今天完成 3 次互动
任务周期：日
统计指标：互动次数
目标数值：3
奖励类型：卡牌
奖励数量：1
分发策略：所有学生
```

达成条件：当天互动次数 >= 3，任务完成，获得 1 张卡牌。次日重置。

---

### 示例 2：每周收集任务

```
任务名称：本周收集 5 种不同卡牌
任务周期：周
统计指标：不同卡牌数
目标数值：5
奖励类型：盲盒
奖励数量：1
分发策略：所有学生
```

达成条件：本周收集到 5 种不同卡牌（去重计数），任务完成，获得 1 个盲盒。下周重置。

---

### 示例 3：里程碑任务（一次性）

```
任务名称：连续互动满 7 天
任务周期：一次性
统计指标：连续互动天数
目标数值：7
奖励类型：卡牌
奖励数量：3
分发策略：所有学生
```

达成条件：连续互动天数 >= 7，任务完成，获得 3 张卡牌。终身只完成一次，不重置。

---

### 示例 4：新学生引导任务

```
任务名称：完成第一次互动
任务周期：一次性
统计指标：互动次数
目标数值：1
奖励类型：卡牌
奖励数量：2
分发策略：新学生默认分配
```

达成条件：完成 1 次互动即达成，仅对首次建档学生分配。
